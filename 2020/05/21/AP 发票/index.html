ORACLE EBS 发票接口

发表于 2020-05-21 | 分类于 ORACLE EBS
字数统计: 1,887 字 | 阅读时长 ≈ 9 分钟

ORACLE EBS API

AP 发票接口

接口功能

  1. 从外围系统导入发票到ORACLE EBS系统
  2. 从外围系统对ORACLE EBS中可用状态的发票进行核销(R12新增)

开发参考

目标表
表名 说明
AP_INVOICES_ALL 发票头表
AP_INVOICE_LINES_ALL 发票行表
AP_INVOICE_DISTRIBUTIONS_ALL 发票分配表，根据行上的会计科目进行汇总
并发程序
标题 路径 内容
职责 / 应付款系统管理员
功能 发票/录入/发票 创建、验证发票，为发票创建会计科目，传输发票至总帐
并发 应付款管理系统开放接口导入 把发票接口表里的数据传送至EBS发票表
并发 发票验证 验证发票内容是否正常
文档 https://docs.oracle.com/cd/E18727_01/doc.121/e12797/T295436T366808.htm#I_prepayx2Dopen ORACLE EBS 应付用户手册
文档 https://docs.oracle.com/cd/E18727_01/doc.121/e12796/T434602T367248.htm#I_ox2Dacctolay ORACLE EBS 开放接口手册
接口信息
表名 说明
ap_invoices_interface 发票头表，生成的信息对应AP_INVOICES_ALL
ap_invoice_lines_interface 发票行表，记录会计科目、金额、会计日期等，生成的信息对应AP_INVOICE_LINES_ALL和AP_INVOICE_DISTRIBUTIONS_ALL
ap_interface_rejections 记录发票错误信息
发票头错误，parent_table为AP_INVOICES_INTERFACE，parent_id关联ap_invoices_interface的invoice_id
发票行错误，parent_table为AP_INVOICE_LINES_INTERFACE，parent_id关联ap_invoice_lines_interface的INVOICE_LINE_ID

字段 说明
last_update_date 更新日期
last_updated_by 更新人
creation_date 创建日期
created_by 创建人
last_update_login 最后登录id，取自fnd_global.login_id
request_id 当前请求id，取自fnd_global.conc_request_id
invoice_id 发票id，取自ap_invoices_interface_s.nextval
invoice_num 发票编号
invoice_type_lookup_code 发票类型，fnd_lookup_values.LOOKUP_TYPE = ‘INVOICE TYPE’
CREDIT类型的发票为红冲发票，金额要为负数
冲销时针对供应商而不是单张发票
invoice_date 发票日期
gl_date 总帐日期
vendor_id 供应商头id，必须为启用的供应商
vendor_site_id 供应商行id，必须为启用的供应商地点
invoice_amount 发票金额
invoice_currency_code 发票币种，如果不是CNY，则exchange_rate_XXX等的三个字段必填
source 来源，随便给
group_id 组id，随便给，也可以同invoice_id
org_id 业务实体id
exchange_rate_type 如果invoice_currency_code不是CNY则必填
exchange_date 同上
exchange_rate 同上
payment_method_code 付款方法
description 备注
terms_id 到期日id，来自ap_terms.term_id
pay_group_lookup_code 支付组，不同的外围系统可以给不同的支付组
fnd_lookup_values.LOOKUP_TYPE = ‘PAY GROUP’
prepay_num 核销字段，核销发票号，发票必须为可用状态
prepay_apply_amount 核销金额

org_id 业务实体id

line_number 发票行号
