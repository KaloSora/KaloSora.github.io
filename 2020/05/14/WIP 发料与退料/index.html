接口信息
表名 说明
mtl_transactions_interface 事务处理主要接口表，记录物料详细信息，以及处理的结果
mtl_transaction_lots_interface 如果物料启用了批次控制，那么要在这个表中插入数据
mtl_serial_numbers_interface 如果物料启用了序列号，那么要在这个表中插入数据
程序要点
1. 事务处理接口表

mtl_transactions_interface

字段 说明
transaction_interface_id 事务处理id，来自序列mtl_material_transactions_s.nextval
transaction_header_id 事务处理标示，建议使用批处理的方式，即每一次提交接口，这个id都保持一致。来自序列mtl_material_transactions_s.nextval
transaction_type_id 事务处理类型id，来自mtl_transaction_types.transaction_type_id。
如果是投料，则mtl_transaction_types.transaction_type_name = ‘WIP 组件发放’；
如果是退料，则mtl_transaction_types.transaction_type_name = ‘WIP 组件退货’；
transaction_action_id 来自mtl_transaction_types.transaction_action_id,跟随事务处理类型
transaction_source_type_id 来自mtl_transaction_types.transaction_source_type_id，跟随事务处理类型
organization_id 库存组织，可参考hr_organization_units.organization_id或fnd_profile.value(‘MFG_ORGANIZATION_ID’)
subinventory_code 子库存，可参考mtl_secondary_inventories.SECONDARY_INVENTORY_NAME
locator_id 货位，可参考mtl_item_locations.INVENTORY_LOCATION_ID
inventory_item_id 物料id，跟随工单的子装配件
transaction_quantity 事务处理数量，如果是出库或子库存转移则为负数，要*-1，如果是入库则为正数。此处退料为正数，发料为负数
transaction_uom 事务处理单位，跟随mtl_system_items_b.PRIMARY_UOM_CODE ，如果单位不一致会报错
operation_seq_num 工序序号，来自wip_requirement_operations.operation_seq_num
transaction_source_id 事务处理来源id，关联其他模块的数据。
账户别名杂项，用别名ID即disposition_id
WIP物料事务，用任务ID即wip_entity_id
SO物料事务，用mtl_sales_orders.sales_order_id子库转移或组织间转移，为空
此处用wip_entities.wip_entity_id
transaction_source_name 事务处理来源名称，此处用wip_entities.wip_entity_name
transaction_reference 事务处理参考，如果是外部系统的数据，建议保存外部系统的id用于追溯，当然也可以存放备注等信息
transaction_date 事务处理日期
transaction_mode 给3
2 Concurrent, Process transaction interface不处理，需要程序调用Inventory transaction worker处理
3 Background, 由Process transaction interface处理
creation_date who字段，创建日期
created_by who字段,创建人
last_update_date who字段，最后更新日期
last_updated_by who字段，最后更新人
last_update_login who字段，来自fnd_global.login_id
process_flag 给1。1 Yes；2 No；3 Error
source_code 来源代码，随便给
source_header_id 来源头id，随便给，这里同transaction_interface_id
source_line_id 来源行id，随便给，这里同transaction_interface_id
2. 批次处理接口表

mtl_transaction_lots_interface

字段 说明
transaction_interface_id 关联事务处理id，同mtl_transactions_interface.transaction_interface_id
lot_number 批次号
source_line_id 来源行id，随便给，这里同transaction_interface_id
transaction_quantity 事务处理数量，同mtl_transactions_interface.ransaction_quantity
primary_quantity 事务处理数量，同mtl_transactions_interface.ransaction_quantity
